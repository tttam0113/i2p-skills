# UI Registry

No UI components have been built for this repository.

This file records the intended registry behavior for `i2p-style`.

## Registry Behavior

`i2p-style` should be able to:

- Establish a style baseline for an existing project.
- Capture reusable patterns after a unit is completed.
- Record deviations that should be fixed later.
- Support software UI patterns and non-code editorial/layout patterns.

## Software Registry Entry Shape

```markdown
### [Component or Pattern Name]

Last updated: [date]

| Property | Value |
| --- | --- |
| Background | [token/class] |

[Short notes]
```
