# UI Tokens

This repository does not currently ship an application UI. This file exists because the software profile follows the starter-kit context structure.

For this repo, "style" means the reusable visual/editorial style artifacts that `i2p-style` will generate for target projects.

## Skill Pack Tokens

Use these naming concepts when designing `i2p-style` templates:

| Concept | Software Output | Non-Code Output |
| --- | --- | --- |
| Visual tokens | `ui-tokens.md` | `style.md` or `layout-guide.md` |
| Pattern rules | `ui-rules.md` | `editorial-style.md` |
| Pattern registry | `ui-registry.md` | `style.md` or `review-checklist.md` |

## Invariants

- Do not invent app UI tokens for this repository.
- Do include template sections that help target projects define their own tokens.
- Keep software and non-code style concepts parallel where possible.
